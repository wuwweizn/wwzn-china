## 关于

星际之门 · StarGate - Green 专版是专为 **Home Assistant Green** 硬件设备优化的网络加速插件，基于首选版本构建，针对 Green 设备的硬件特性和系统环境进行了适配优化。功能与首选版本一致，支持对 HA 主机及 Docker 网络内所有服务同时加速，并可为局域网内其他设备提供网页访问代理，实现类似旁路由的效果。

**支持架构：** `amd64` / `aarch64`

**适用设备：** Home Assistant Green（及其他官方 HAOS 设备）

---

## 与首选版本的区别

| 对比项 | 首选版本 | Green 专版（本插件）|
|--------|---------|-------------------|
| 适用设备 | 通用 HAOS 设备 | 针对 HA Green 优化 |
| 功能特性 | 完整 | 完整，与首选版本一致 |
| 底层适配 | 通用配置 | 针对 Green 硬件环境适配 |

> 如果你使用的是 **Home Assistant Green** 设备，建议优先使用本版本；其他设备请使用 [星际之门（首选）](../stargate/)。

## 使用须知

- **仅支持官方 HAOS 系统**，haos 系列均可使用；网络底层经过加强，支持类似旁路由的功能。
- 此插件只能在 HA **完全启动、服务配置完成后**才能生效，对首次安装无法提供加速。
- 本插件需要**配合第三方代理订阅**使用（v2ray / xray 订阅链接）。
- 由于各地 ISP 网络环境不同，特定地区可能无法正常使用。
- **注意：** 本版本对 VLESS+REALITY 等较新协议支持有限，如有需要请使用 [星际之门 · StarGate（备选）](../stargate2/)。

## 首次配置与使用

### 第一步：启动插件

在 HA 插件页面启动本插件（此时代理服务尚未运行）。

### 第二步：进入 Web 控制台

在浏览器打开：

```
http://<你的HA主机IP>:2017
```

### 第三步：初始化设置

1. **注册用户名和密码**（请妥善保存，忘记后需清除数据重置）
2. 进入顶部菜单「**设置**」
3. 关闭「开启 IP 转发」
4. 打开「开机端口分享」
5. 点击左上角「**启动**」按钮，启动代理服务

### 第四步：导入订阅

在控制台中导入你的 v2ray / xray 订阅链接，选择节点后即可使用（支持多节点负载均衡）。

> 更多进阶功能（分流规则、负载均衡等），请参考 [v2rayA 官方文档](https://v2raya.org/docs/prologue/introduction/) 自行配置。

## 启动后验证

在 HA 终端（Terminal 插件）中运行：

```bash
# 查看当前出口 IP 信息
curl ipinfo.io

# 测试目标网站的 HTTP 响应状态码
curl -sL -o /dev/null -w "%{http_code}" https://www.google.com
```

如果出口 IP 已变为代理节点 IP，说明加速已成功生效。

## 常见问题

**Q：我有 HA Green，用这个还是首选版本？**
建议使用本 Green 专版，针对 Green 硬件做了专项适配，兼容性更好。

**Q：插件启动了但无法访问 2017 端口？**
确认 HA 已完全启动，且本插件状态为"运行中"。检查路由器或防火墙是否放行了 2017 端口。

**Q：重启 HA 后代理失效？**
在插件配置中将启动方式改为"自动"，并确保已在 v2rayA 控制台中开启了「开机端口分享」。

**Q：局域网其他设备也想使用代理？**
在对应设备的网络设置中，将 HTTP 代理地址指向 `<HA主机IP>:20171`（v2rayA 默认 HTTP 代理端口）。
