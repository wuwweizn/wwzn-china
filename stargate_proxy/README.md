## 关于

星际之门 · StarGate - 国内加速版与首选版本功能完全一致，区别在于**插件镜像通过国内加速节点分发**，专为在国内网络环境下无法顺畅拉取 Docker 镜像的用户提供更快速、稳定的安装体验。

如果你在安装首选版本时遇到镜像拉取超时、失败等问题，可以改用本版本。

**支持架构：** `amd64` / `aarch64`

---

## 与首选版本的区别

| 对比项 | 首选版本 | 国内加速版（本插件）|
|--------|---------|-------------------|
| 功能特性 | 完整 | 完整，与首选版本完全一致 |
| 镜像来源 | Docker Hub（国际） | 国内加速节点分发 |
| 适用场景 | 网络环境良好 | 国内网络拉取镜像慢或失败时 |

> **建议：** 优先尝试 [星际之门（首选）](../stargate/)；若安装过程中镜像拉取失败或速度极慢，再使用本版本。

## 使用须知

- **仅支持官方 HAOS 系统**，haos 系列均可使用；网络底层经过加强，支持类似旁路由的功能。
- 此插件只能在 HA **完全启动、服务配置完成后**才能生效，对首次安装无法提供加速。
- 本插件需要**配合第三方代理订阅**使用（v2ray / xray 订阅链接）。
- 由于各地 ISP 网络环境不同，特定地区可能无法正常使用。
- **注意：** 本版本对 VLESS+REALITY 等较新协议支持有限，如有需要请使用 [星际之门 · StarGate（备选）](../stargate2/)。

## 首次配置与使用

### 第一步：启动插件

在 HA 插件页面启动本插件（此时代理服务尚未运行）。

### 第二步：进入 Web 控制台

在浏览器打开：

```
http://<你的HA主机IP>:2017
```

### 第三步：初始化设置

1. **注册用户名和密码**（请妥善保存，忘记后需清除数据重置）
2. 进入顶部菜单「**设置**」
3. 关闭「开启 IP 转发」
4. 打开「开机端口分享」
5. 点击左上角「**启动**」按钮，启动代理服务

### 第四步：导入订阅

在控制台中导入你的 v2ray / xray 订阅链接，选择节点后即可使用（支持多节点负载均衡）。

> 更多进阶功能（分流规则、负载均衡等），请参考 [v2rayA 官方文档](https://v2raya.org/docs/prologue/introduction/) 自行配置。

## 启动后验证

在 HA 终端（Terminal 插件）中运行：

```bash
# 查看当前出口 IP 信息
curl ipinfo.io

# 测试目标网站的 HTTP 响应状态码
curl -sL -o /dev/null -w "%{http_code}" https://www.google.com
```

如果出口 IP 已变为代理节点 IP，说明加速已成功生效。

## 常见问题

**Q：首选版本能用，还需要装这个版本吗？**
不需要，两者功能完全相同，同时安装运行会产生端口冲突。选其中一个使用即可。

**Q：这个版本和首选版本可以同时安装吗？**
不建议，两者都使用 2017 端口，同时运行会冲突。

**Q：重启 HA 后代理失效？**
在插件配置中将启动方式改为"自动"，并确保已在 v2rayA 控制台中开启了「开机端口分享」。

**Q：插件日志占用磁盘空间过大？**
在插件配置中开启「关闭部分日志」选项，平时无需查看日志时建议保持开启状态。
